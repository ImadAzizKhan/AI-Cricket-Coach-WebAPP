# 🏏 OmniComm: Smart Bat & AI Cricket Coach

**Author:** Muhammad Imad Aziz Khan  
**Context:** Developed as a Human-Computer Interaction (HCI) project at Sapienza University of Rome.

This project merges Computer Vision, state-of-the-art Machine Learning, IoT hardware, and an interactive web application to create a real-time, closed-loop AI cricket coach. It uses MediaPipe for pose estimation, an optimized XGBoost classifier trained on 36 kinematic features, a Flask web server for mobile interaction, and communicates via UDP to an ESP32 micro-controller embedded in a physical smart cricket bat to deliver haptic feedback.

## ✨ Key Features

* **Advanced ML Pipeline:** Features an Extreme Gradient Boosting (XGBoost) classifier optimized with Optuna and SMOTE. The model extracts 36 biomechanical "delta" features (e.g., shoulder tilt, wrist-to-hip distance, trunk lean) across 3 phases of the swing for high-accuracy classification.
* **Voice-Activated HCI:** Integrated Native Browser Speech Recognition allows batters to control the UI completely hands-free ("Start Practice", "Reset") while wearing gloves or standing away from the screen.
* **Targeted Lesson Mode & Ghost UI:** Select a specific shot (e.g., Cover Drive, Sweep) on the web app. The UI displays an ideal "Ghost Skeleton" for visual reference before you practice.
* **Real-Time Grading & Haptic Feedback:** The system waits for your swing's peak kinematic velocity, grades your specific joint angles against the target shot, and sends a UDP packet to an ESP32 to vibrate the bat handle mid-swing if form errors are detected.

## 📊 The Dataset (CricShot10k)
To train the machine learning model from scratch, this project utilizes the **CricShot10k Dataset**, a comprehensive collection of diverse cricket batting strokes. 

Due to storage limits and licensing, the raw video files are not included directly in this repository. 
1. Open the `1_Raw_Videos` file in this repository to find the download links for the dataset.
2. Extract the downloaded videos into a local folder named `1_Raw_Videos`.
3. Use the provided processing scripts to clean up and flatten the nested folder structures before running the extraction pipeline.

## 🚀 How to Run the Live App
For seamless, zero-latency communication between your Python backend and the web interface, both devices must be on the exact same local network. 

**1. Network Setup**
* Enable your mobile phone's Wi-Fi Hotspot and connect your laptop to this network.

**2. Locate Your IP Address**
* Open PowerShell or Command Prompt on your laptop.
* Execute the `ipconfig` command and note your **IPv4 Address** (e.g., `192.168.x.x`).

**3. Initialize the Server**
* Open a terminal within your project directory.
* Launch the ML-powered backend by running: `python appv2.py`

**4. Connect via Mobile**
* Open a web browser on your phone (Chrome or Safari recommended).
* Enter your laptop's IPv4 address followed by `:5000` (e.g., `http://192.168.x.x:5000`).
* Stand back, say "Start Practice", and swing!

## ⚙️ Hardware Integration & Future VR Scope
This computer vision module is engineered to operate in tandem with a custom physical smart bat, powered by an ESP32 microcontroller and a BNO085 IMU sensor. Ultimately, these combined kinematic data streams will be transmitted via UDP to the Unity game engine, establishing a fully immersive, physics-accurate VR training environment.
